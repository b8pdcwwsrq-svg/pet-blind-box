## 一、产品定位

一款通过**呼吸光球引导正念 + 随机小事探索**的方式，帮助用户**打破惯性生活、觉察情绪、接纳每一种感受**的情绪记录工具。

**核心理念：** 你不需要好起来，你只是来体验的。

---

## 二、目标用户

- 一二线城市年轻人（20-35 岁）
- 感到生活陷入惯性重复，渴望"活着的感觉"
- 对正念 / 心理学 / 身心灵有了解或好奇
- 不是追求"被治愈"，而是想要"感觉到"

### 5.1 App.tsx 全部状态

采用 React Hooks 集中管理，不使用 Redux/Zustand 等外部状态库。

| `isAnimating` | `boolean` | `false` | 动画锁定，阻止并发操作 |

| `isMusicPlaying` | `boolean` | `false` | 背景音乐状态 |
| `orbFlash` | `boolean` | `false` | 光球闪亮动画触发器 |
| `showEnvelope` | `boolean` | `false` | 信纸可见 |
| `envelopeTitle` | `string` | `""` | 信纸标题 |
| `envelopeText` | `string` | `""` | 信纸正文 |
| `envelopeImage` | `string \| null` | `null` | 信纸照片（base64） |
| `breathePhase` | `0\|1\|2` | `0` | 呼吸引导相位 |
| `showWelcome` | `boolean` | `!localStorage.getItem(...)` | 首次引导 |
| `moodFeedbackMsg` | `string` | `""` | 情绪反馈气泡文字 |
| `isDragging` | `boolean` | `false` | 正在拖拽 |
| `dragReady` | `boolean` | `false` | 拖拽已激活 |
| `dragY` | `number` | `0` | 拖拽偏移量（px） |
| `showTimeline` | `boolean` | `false` | 时光轴可见 |

### 5.2 状态机：pageState 流转

```
idle
  │ 点击光球
  ▼
loading (700ms)
  │ 抽取完成
  ▼
revealed
  ├─ 点击光球 → loading (无限次)
  ├─ 【完成小事】→ 情绪选择弹窗 → collected
  └─ 【记录当下】→ 信纸 → 情绪选择弹窗 → collected
        │
        ▼
collected (2500ms 后自动回到 idle)
  └─ 【再拾一段】→ idle
```

### 5.3 ref 使用的内部状态（不触发渲染）

- `audioRef` → 替换为模块级 `let audioInstance`（跨组件实例持久化）
- `cameraInputRef` — `<input type="file">` 引用
- `dragTimerRef` — 拖拽激活延迟计时器
- `dragStartRef` — 拖拽起点坐标 + 时间
- `dragYRef` — 拖拽偏移量（同步写入，避免闭包陈旧值）
- `isDraggingRef` — 拖拽状态（同步写入）

## 六、功能详解

### 6.1 呼吸光球

**实现方式：** 纯 CSS，无 JS 动画循环。
- **尺寸：** 100×100px 圆形
- **视觉效果：** 3 层 radial-gradient + 5 层外发光 box-shadow + 1 层 inset 内发光
- **表面质感：** `::before` 伪元素，SVG fractalNoise 滤镜，mix-blend-mode: soft-light
- **呼吸动画（`@keyframes orb-breathe`）：** 12 秒循环
  - 0% → 33.33%（吸气 4s）：scale 0.80→1.15，opacity 0.5→1，blur 5px→1px，外发光扩张
  - 33.33% → 66.66%（屏息 4s）：scale 保持在 1.10，维持高亮度
  - 66.66% → 100%（呼气 4s）：scale 回到 0.80，恢复低亮度
- **觉醒模式（`orb-awaken`）：** 1 秒循环，用于抽取后的反馈
- **闪亮动画（`orb-flash`）：** 6 秒一次性动画，完成小事后的庆祝效果
- **拖拽模式（`drag-ready` / `dragging`）：** 暂停呼吸动画，box-shadow 增强

### 6.2 核心流程交互

#### 点击光球（handleOrbClick）
1. 检查 `isAnimating` 和 `pageState`
2. 设 `isAnimating = true`，`pageState = "loading"`
3. `getCurrentPeriod()` 获取当前时段
4. 从 `ALL_EVENTS` 中筛选适合此时段的或 "anytime" 的事件
5. 随机选取一个
6. 700ms 后显示抽取结果

#### 拖拽手势
1. **pointerDown：** 记录起点 + 时间，设 200ms 延迟激活拖拽
2. **pointerMove：** 移动超过 8px 取消拖拽（避免误触），激活后只向下拖动（带阻尼）
3. **pointerUp：** 拖动超过 20vh 打开时光轴，否则忽略
4. 短按（<200ms）→ 触发点击逻辑，通过原生 onClick 处理
5. 移动端支持：`touchAction: "pan-y"` 允许垂直滚动

#### 情绪选择（handleMoodSelect）
- 两种路径：
  - **从信纸来：** 有标题/正文/照片 → 存入作为独立记忆条目
  - **直接从光球来：** 只有情绪色 → 结合 currentEvent 存入
- 保存到 localStorage + state，600ms 后显示 collected 状态
- 2500ms 后自动回到 idle

### 6.3 情绪系统

- 7 种情绪：calm（平静）、warm（温暖）、happy（开心）、content（满足）、sad（低落）、tired（疲惫）、confused（困惑）
- `ALL_MOODS` 定义在 `memoryUtils.ts`
- 每种情绪对应颜色和 2 条反馈文案
- **注意：** `confused` 情绪缺少反馈文案（`MOOD_FEEDBACK` 中未定义）

### 6.4 小事事件系统

- 50 件治愈小事，每个事件包含：id, emoji, text, keyword, period[], location
- period 分类：morning / afternoon / evening / night / anytime
- 按当前时段筛选，若不足 3 条则放宽限制
- 关键词用于时间轴标签显示

### 6.5 信纸记录

- 底部浮现的信纸式输入界面
- 标题 + 正文（行高 32px，信纸横线背景）
- 照片输入（capture="environment" 调用相机）
- ✕ 关闭按钮
- 提交 → 情绪选择弹窗 → 存入

### 6.6 时光印记（时间轴）

**FuguangTimeline 组件：**

- 全屏 fixed 容器，z-index: 100
- 头部：图标 + "时光印记" + 条目计数 + ✕ 关闭
- **玻璃罐萤火虫（GlassJar 子组件）：**
  - `<canvas>` 绘制玻璃罐 + 萤火虫发光点
  - 按月切换（← → 箭头）
  - 每个有数据的日期 = 一个萤火虫，颜色 = 该日最后一条记录的情绪色
  - requestAnimationFrame 循环：萤火虫缓慢浮动 + 呼吸发光
  - 点击萤火虫可跳转到对应日期的条目
- **时间轴列表：**
  - 按日期分组，显示"今天 / 昨天 / x月x日"
  - 每条记录：左侧情绪色水彩圆点 + 时间，右侧关键词 + 正文 + 照片
  - 圆点间有连接线（:before / :after 伪元素）
  - **滑动删除：** 左滑 >55px 显示红色删除按钮，支持 pointer/touch 事件
  - Framer Motion 入场动画

### 6.7 四季轻音乐

- `SEASON_MUSIC` 按 spring/summer/autumn/winter 定义曲库
- `getSeason()` 根据月份判断季节
- 使用模块级 `let audioInstance` 确保音频对象在组件重新挂载时保持
- 音量 0.25，自动循环
- 音符按钮在播放时有 pulse 动画

### 6.8 首次引导

- 检查 `localStorage.getItem("fuguang-visited")`
- 未访问过 → 显示全屏遮罩+毛玻璃卡片
- 文案："不需要好起来，只需要感觉到"
- 点击"开始"后存入 localStorage + 关闭

## 七、数据层

### 7.1 核心类型定义（memoryUtils.ts）

```typescript
// 小事事件
interface GlowEvent {
  id: number;
  emoji: string;
  text: string;
  keyword: string;
  period: string[];      // "morning" | "afternoon" | "evening" | "night" | "anytime"
  location: string;
}

// 记忆条目（localStorage 中持久化）
interface MemoryEntry {
  id: number;
  date: string;          // "6月26日"
  month: number;         // 1-12
  time: string;          // "14:30"
  emoji: string;
  eventText: string;
  keyword: string;
  moodId: string;
  moodLabel: string;
  moodColor: string;     // 十六进制颜色
  response: string;      // 用户回应（暂未使用）
  imageData?: string;    // base64 图片数据
  location: string;
}
```

### 7.2 localStorage 键值

| Key | 值类型 | 说明 |
|---|---|---|
| `fuguang-memory` | `MemoryEntry[]` JSON | 所有记忆条目 |
| `fuguang-visited` | `"true"` 或缺失 | 首次引导标记 |

### 7.3 无后端架构

所有数据存储在浏览器 localStorage 中。优点是：
- 零服务器成本
- 用户隐私（数据不上传）
- 离线可用

限制：
- 单设备数据（无云同步）
- 存储上限约 5-10MB
- 清除浏览器数据会丢失

## 九、已知问题

1. **首页布局重叠：** 光球下方的提示文字（"点一下，做一件小事"、"吸气…"、"↓ 继续下拉"）和"记录当下"按钮在部分屏幕高度下重叠。根源是光球用了 `position: absolute` + 固定偏移的提示文字改为流式布局后，按钮区域计算未更新。

2. **CSS 文件过大：** index.css 约 2342 行，包含大量未使用的样式（如 `.ink-layer`、`.oil-glass-layer`、`.fuguang-landscape` 等的鹅卵石大地艺术样式是旧版本遗留）。建议按组件拆分。

3. **信纸关闭按钮：** 样式硬编码为 inline style，应提取为 CSS 类。

4. **节气拖拽提示文字：** 节气卡片下方的拖拽提示文字位置用了 `position: relative; bottom: -4px` 的 inline style，在部分设备上显示异常。

5. **FuguangTimeline canGoPrev/Next：** 月份切换逻辑有边界情况，当前月不能往后切的判断不完整。

6. **confused 情绪缺少反馈文案：** `MOOD_FEEDBACK` 对象中未包含 `confused` 条目，选择后情绪反馈气泡将是空的。

7. **无 PWA 支持：** 缺少 manifest.json 和 service worker，无法安装到手机桌面。

8. **vite.config.ts 未使用：** 存在 `vite.config.js` 和 `vite.config.ts` 两个配置文件，后者未被实际使用（Vite 优先读取 .js）。

9. **节气日期硬编码：** `SOLAR_TERMS` 中的日期范围为近似值（如立春 2/3-5），与实际天文节气可能有 1-2 天偏差。

- Vite 自动热更新，修改代码后即时生效
- 所有数据存储在浏览器 localStorage 中，清除浏览器数据会丢失测试记录
- `.gitignore` 已包含 `node_modules/` 和 `dist/`
