# 今日拾光 — 碎片功能产品设计 v1.0

## 核心理念

**收集大家的美好的地方。**

不是社交，不对话，不回复。每个人留一片碎片（一句话 + 可选贴纸），像在黑暗里放一盏灯，你不知道谁看见了，但灯亮着就够了。看到别人的碎片心有共鸣，也可以"也留一片"，独立进池，不绑定原文，不回复。

## 双线并行架构

整个 App 从此分为两条互相独立的线：

| 维度 | 🧩 做一件小事（行动线） | 🪶 拾一片碎片（感受线） |
|------|-----------------------|-----------------------|
| 来源 | 80 条预设祈使句 | 预设陌生人短句 + 用户自创碎片 |
| 语气 | "拉开窗帘……" "给自己泡杯茶……" | "今天看到一只猫蹲在路灯下……" |
| 入口 | 「记录当下」按钮 | 🪶 独立按钮 |
| 交互 | 完整信纸（标题+正文+照片+贴纸+心情） | 轻量卡片（一句话+可选贴纸） |
| 提交 | 选心情→存入时光轴 | 直接放下，不选心情 |
| 每天次数 | 不限 | 写 1 次，抽不限 |

## 碎片功能详细设计

### 入口
首页主界面，在"记录当下"按钮旁边增加 🪶 按钮。
- 如果今天还没写过碎片 → 正常显示
- 如果今天已经写过 → 按钮变半透明，点击提示"今天的光已经拾过啦，明天再来"

### 写碎片流程
1. 点击 🪶 → 弹出轻量卡片
2. 输入框："今天想留下一句什么？"（最多 50 字）
3. 可选：点击 🎀 从贴纸库选一个贴纸附上
4. 点击「放下」→ 存入碎片池，弹出"已拾取一片光"，2 秒消失
5. 不选心情，不选标题

### 抽碎片流程
1. 点击主界面的光球/节气卡片 → 抽取碎片
2. 碎片池混合：预设陌生人短句 + 自己写的碎片（如果已开放多人则包含别人的碎片）
3. 抽到的碎片显示在卡片上，下方显示：
   - 「也留一片」按钮 → 打开写碎片卡片，自动清空输入框
   - 如果是自己写的碎片 → 不标记，但不小心看笔迹会认出来
4. 每天抽不限次数，但写只有 1 次

### 碎片列表
时光轴新增 tab 切换：「📝 记录 / 🪶 碎片」
- 「记录」tab：展示原有的日记条目
- 「碎片」tab：展示自己写过的碎片历史，按时间倒序

### 引用碎片
- **写碎片时**：不引用。看到别人的碎片心有所感，写下的内容独立进池，不与原碎片绑定。不是为了回复，而是为了"也留下一盏灯"。
- **写日记时**：可以引用。受碎片触动后打开"记录当下"，在正文中自然引用碎片内容。这是"碎片改变了我的生活"，是日记的自然组成部分。

### 与拍照转贴纸的联动
- 写碎片时可以从贴纸库里选已经做好的贴纸附上
- 拍照→转贴纸的流程不变，贴纸库是共享的

## 碎片池说明（v1.0 为本地方案）

### 碎片来源
1. **预设碎片** — 硬编码在代码中，30-50 条"陌生人语气"的短句
2. **用户自创碎片** — 用户写的存入 localStorage

### 抽卡逻辑
`预设碎片 + 自己写的碎片` 混合随机抽取。

### 容量
纯前端 localStorage，单条碎片存储量极小（一句话的文本 + 可选贴纸 ID），预估 0.5KB/条。每天写 1 条，一年约 365 条 ≈ 180KB，完全无压力。

## 后续可能的开放路径（暂不实现）

1. localStorage 共享池（通过 GitHub Gist / Cloudflare KV 同步）
2. 碎片展示"嗯"计数（看到别人按了"也留一片"后我的碎片被抽到次数）
3. 碎片池容量管理策略
