# 今日拾光 — 产品评估报告

> 生成日期：2026-06-27
> 评估范围：竞品对比 / UI 设计评估 / 每日签文方案评估

## 一、竞品对比评估

### 1.1 直接竞品

| 产品 | 定位 | 核心机制 | 情绪态度 | 商业化 |
|---|---|---|---|---|
| **今日拾光** | 正念引导 + 情绪记录 | 呼吸光球 → 小事抽取 → 情绪标记 | 中性（每种情绪都允许） | 未商业化 |
| **Mooda** | 情绪日记 | 选心情 → 写日记 → 看统计 | 偏积极 | 免费 + 高级会员 |
| **Daylio** | 微观日记 | 选活动+心情 → 数据统计 | 偏积极 | 免费 + 高级会员 |
| **Gratitude** | 感恩日记 | 每日写三件感恩的事 | 积极导向 | 免费 + 会员 |
| **I Am** | 正向 affirmations | 每日推送肯定语句 | 积极导向 | 免费 + 会员 |
| **小睡眠** | 助眠 + 白噪音 | 声音库 + 冥想引导 | 中性 | 免费 + 会员 + 内容付费 |
| **潮汐** | 冥想 + 专注 | 番茄钟 + 冥想引导 | 中性 | 免费 + 会员 |

### 1.2 功能对标矩阵

```diff
功能                    今日拾光    Mooda    Daylio    潮汐
───                     ───────    ─────    ──────    ────
情绪记录                ✅          ✅       ✅        ❌
呼吸引导                ✅          ❌       ❌        ✅

白噪音/音乐             ✅          ❌       ❌        ✅
图片/文字日记           ✅          ✅       ✅        ❌
24节气                 ✅          ❌       ❌        ❌
社交/分享               ❌          ❌       ❌        ❌
```

### 1.3 差异化优势

今日拾光最独特的地方在于**不对情绪做价值判断**。Daylio、Mooda 本质上还是在引导用户"追求好心情"，而今日拾光的理念是"每种情绪都可以"。

另一个差异点是**交互方式**——呼吸光球点击抽取盲盒式小事，比标准的"选心情→写日记"流程更有趣味性和仪式感。

### 1.4 竞争力弱点

1. **无数据可视化**：竞品都有周报月报，今日拾光只有时间轴列表
2. **无推送**：竞品通过每日推送维持活跃
3. **内容深度不足**：潮汐有完整的冥想课程体系，今日拾光只有随机小事
4. **无社交裂变**：没有分享、邀请机制

## 二、UI 设计评估

### 2.1 整体设计评价

**亮点：**
- 色调统一（暖米色 #f7f4ed + 莫兰迪点缀），氛围感强
- 霞鹜文楷字体独特，与"东方留白"定位一致
- 呼吸光球的 CSS 实现精致，多层发光 + 噪声纹理质感好
- 情绪色球高光渐变处理细腻
- 玻璃罐萤火虫 Canvas 动画超出预期

**问题：**

| 问题 | 严重程度 | 说明 |
|---|---|---|
| 首页布局重叠 | 高 | 光球下方提示文字和按钮挤在一起 |
| 无适配响应式 | 中 | 固定 max-width 420px，大屏留白太多 |
| 交互反馈弱 | 中 | 点击/切换没有触感反馈（移动端） |
| 加载状态简陋 | 低 | 只有 3 个弹跳圆点 |
| 情绪选择弹窗 | 中 | 从底部弹出但无动画过渡 |
| 信纸样式 | 中 | 横线背景在某些设备上对不齐 |

### 2.2 具体改进建议

**高优先级：**
1. 光球 + 提示文字 + 按钮三层用 flex gap 排列，去掉 absolute 定位
2. 情绪选择弹窗加 Framer Motion 底部滑入动画
3. 信纸横线背景改为 `repeating-linear-gradient` 确保对齐

**中优先级：**
4. 加一个 PWA 的 splash screen
5. 信纸关闭按钮从 inline style 提取为 CSS 类
6. 事件卡片显示位置标签（现有数据有 location 字段但未使用）

**低优先级：**
7. 光球拖拽加入震动反馈（已有 navigator.vibrate，但只在桌面测试）
8. 节气卡片季节色和光球呼吸联动

## 三、每日签文方案评估

### 3.1 方案概述

在现有节气卡片位置增加"每日签文"，包括：
- 一句经典语录（与神对话 / 当下的力量）
- 一个"宜"字指引标签
- 可点击刷新换一条

### 3.2 可行性评估

| 维度 | 评分 | 说明 |
|---|---|---|
| 开发成本 | ★★★★★ 极低 | 纯前端，数据在 memoryUtils.ts 加一个数组即可 |
| 设计兼容性 | ★★★★☆ | 节气卡片位置已存在，直接复用卡片样式 |
| 用户体验 | ★★★★★ | 增加每天打开的动力，和产品调性一致 |
| 商业价值 | ★★★☆☆ | 是内容沉淀，但不是直接付费点 |

### 3.3 推荐实现方式

**数据层：** 在 `memoryUtils.ts` 新增 `DAILY_QUOTES` 数组

```typescript
interface DailyQuote {
  text: string;        // 语录正文
  source: string;      // 出处（"与神对话" / "当下的力量"）
  yi?: string;         // "宜"字标签（如"宜静坐"、"宜发呆"）
}

const DAILY_QUOTES: DailyQuote[] = [
  { text: "你不需要被修复。你从来都没有破碎过。", source: "与神对话", yi: "宜存在" },
  { text: "过去和未来都是大脑对自己的幻觉和欺骗。", source: "当下的力量", yi: "宜当下" },
  // ... 30-50 条
]
```

**展示层：** 修改 App.tsx 中节气卡片的渲染逻辑：

```
如果是节气日　→ 显示节气卡片（现有逻辑不变）
如果是普通日　→ 显示"今日签文"卡片（卡片样式复用）
               ├─ 顶部：当月 + 节气信息
               ├─ 中间：语录 + 出处
               ├─ 底部："宜"字标签
               └─ 右下角：🔄 换一条按钮
```

**出现逻辑：**
- 每天首次打开 → 固定一条（根据日期 seed 决定）
- 手动刷新 → 随机换一条（不改变每日固定签）
- idle 状态点击光球抽取小事后，签文保留在背景

### 3.4 语录来源建议

建议分三个池子，总共 40-60 条：

**池 A（与神对话系，15-20 条）：**
- "你不需要被修复。你从来都没有破碎过。"
- "你不是来学习的，你是来回忆的。"
- "所有的发生，都是完美发生在完美的时间里。"

**池 B（当下的力量系，15-20 条）：**
- "当你接受当下的现实，你就在那一刻超越了它。"
- "观察你的思绪，而不被它带走。"
- "过去没有任何事情可以阻止你活在当下。"

**池 C（日常治愈系，10-20 条）：**
- "阳光落在皮肤上的温度，就是此刻的全部。"
- "慢慢呼吸，就是一个小小的奇迹。"
- "今天什么都不用做，存在就够了。"

### 3.5 风险与注意事项

1. **不要变成"鸡汤"**：语录要保持"中性接纳"的基调，不要写成"加油""你能行"那种正向激励。一旦变成鸡汤就和产品核心理念冲突了。
2. **不要保证"好运"**：你之前提到的好运文字，建议放在节气限定里（比如立春说"万物复苏"），不要做成每天抽好运签，会偏离产品定位。
3. **控制刷新频率**：每天自动换一条即可，不要每次打开都换，失去了"每日"的仪式感。
4. **商业化顾虑**：语录本身不能直接收费，但每日签文可以作为"会员可解锁更多语录池"的基础功能点。

### 3.6 结论

| 评估项 | 结论 |
|---|---|
| 做不做 | ✅ 建议做，成本和风险都很低 |
| 优先级 | 中（在 Phase 2 布局修复之后） |
| 开发时间 | 约 1-2 小时 |
| 先做哪个池 | 先从池 A + 池 B 开始（30-40 条），和产品核心理念最契合 |
